## 📝 테스트 시나리오

### 시나리오 1: 자사제품 등록

**목표:** 자사제품 코드(LCS) 자동 발부 확인

**단계:**
1. 제품 정보 페이지 접속
2. "신규 등록" 버튼 클릭
3. 다음 정보 입력:
   - 제품명: `프리미엄 더치커피 800ml`
   - 제품구분: `자사제품` (자동으로 선택됨)
   - 카테고리: `더치커피`
   - 판매단가: `15,000`
   - 원가: `8,000`

4. 서류관리에서 다음 항목 체크:
   - ☑ 시험성적서
   - ☑ 원산지증명서

5. "저장" 클릭

**기대 결과:**
- ✅ 제품코드: `LCS-001` 자동 생성
- ✅ 제품 목록에 추가됨
- ✅ 서류 상태: `2/5 입수` 표시

---

### 시나리오 2: 수입제품 등록

**목표:** 수입제품 코드(LCI) 자동 발부 확인

**단계:**
1. "신규 등록" 버튼 클릭
2. 제품구분 드롭다운에서 `수입제품` 선택
3. 다음 정보 입력:
   - 제품명: `에티오피아 원두 1kg`
   - 카테고리: `원두커피`
   - 판매단가: `45,000`
   - 원가: `25,000`

4. 서류관리에서 다음 항목 체크:
   - ☑ 시험성적서
   - ☑ 원산지증명서
   - ☑ 수입신고필증
   - ☑ 거래명세서

5. "저장" 클릭

**기대 결과:**
- ✅ 제품코드: `LCI-001` 자동 생성 (수입제품이므로 LCI 프리픽스)
- ✅ 제품 목록에 추가됨
- ✅ 서류 상태: `4/5 입수` 표시

---

### 시나리오 3: OEM제품 등록

**목표:** OEM제품 코드(LCO) 자동 발부 확인

**단계:**
1. "신규 등록" 버튼 클릭
2. 제품구분 드롭다운에서 `OEM제품` 선택
3. 다음 정보 입력:
   - 제품명: `A사 커스텀 블렌드 500ml`
   - 제조업체/OEM업체: `A사`
   - 카테고리: `음료`
   - 판매단가: `12,000`
   - 원가: `6,000`

4. 서류관리에서 다음 항목 체크:
   - ☑ 거래명세서

5. "저장" 클릭

**기대 결과:**
- ✅ 제품코드: `LCO-001` 자동 생성 (OEM제품이므로 LCO 프리픽스)
- ✅ 제품 목록에 추가됨
- ✅ 서류 상태: `1/5 입수` 표시

---

### 시나리오 4: 기타제품 등록 (기재 내용 포함)

**목표:** 기타제품 코드(LCE) 및 기타 서류 기재 기능 검증

**단계:**
1. "신규 등록" 버튼 클릭
2. 제품구분 드롭다운에서 `기타제품` 선택
3. 다음 정보 입력:
   - 제품명: `특별 한정판 세트`
   - 카테고리: `기타`
   - 판매단가: `50,000`
   - 원가: `30,000`

4. 서류관리에서 다음 항목 체크:
   - ☑ 기타 (입력 필드에 "한정판 판매 승인서" 기재)

5. "저장" 클릭

**기대 결과:**
- ✅ 제품코드: `LCE-001` 자동 생성 (기타제품이므로 LCE 프리픽스)
- ✅ 제품 목록에 추가됨
- ✅ 서류 상태: `1/5 입수` 표시
- ✅ 기타 서류 기재 내용 저장됨

---

### 시나리오 5: 제품 수정 및 서류 추가

**목표:** 기존 제품 수정 및 서류 상태 업데이트 확인

**단계:**
1. 시나리오 1에서 등록한 `프리미엄 더치커피 800ml` 제품의 "수정" 버튼 클릭
2. 기존 체크 상태 확인:
   - ☑ 시험성적서 (체크됨)
   - ☑ 원산지증명서 (체크됨)
   - ☐ 수입신고필증 (미체크)
   - ☐ 거래명세서 (미체크)
   - ☐ 기타 (미체크)

3. 추가 항목 체크:
   - ☑ 수입신고필증
   - ☑ 거래명세서

4. "저장" 클릭

**기대 결과:**
- ✅ 제품코드 유지: `LCS-001`
- ✅ 제품 목록에서 서류 상태 업데이트: `4/5 입수`
- ✅ 수정 완료 메시지 표시

---

### 시나리오 6: 제품구분 필터링

**목표:** 제품구분별 필터링 기능 확인

**단계:**
1. 제품 목록 페이지에서 "제품구분 전체" 드롭다운 클릭
2. `수입제품` 선택
3. 목록 확인

**기대 결과:**
- ✅ LCI 코드 제품만 표시됨
- ✅ KPI 카드에서 "수입제품" 수 정확함
- ✅ 다른 제품구분 제품은 숨겨짐

**추가 테스트:**
- 각 제품구분별로 필터링 반복 (자사제품, OEM제품, 기타제품)
- "제품구분 전체" 선택 시 모든 제품 표시 확인

---

### 시나리오 7: 검색 기능

**목표:** 제품명/코드 검색 기능 확인

**단계:**
1. 검색창에 `LCI` 입력
2. 결과 확인

**기대 결과:**
- ✅ LCI 코드 제품만 표시됨

**추가 테스트:**
- 제품명으로 검색 (예: "더치커피")
- 부분 검색 (예: "커피")
- 존재하지 않는 제품 검색

---

### 시나리오 8: 팀원 간 데이터 공유

**목표:** 실시간 데이터 동기화 확인

**단계:**
1. **사용자 A**: 로컬 테스트 서버에서 새 제품 등록
2. **사용자 B**: 다른 브라우저/기기에서 같은 URL 접속
3. 사용자 B의 화면에서 사용자 A가 등록한 제품 확인

**기대 결과:**
- ✅ 사용자 B의 화면에 즉시 새 제품 표시 (1-2초 이내)
- ✅ 제품 정보 완전히 일치
- ✅ 서류 상태도 동일하게 표시

---

### 시나리오 9: 오프라인 상태 확인

**목표:** 오프라인 상태 감지 및 알림 확인

**단계:**
1. 제품 정보 페이지 접속
2. 브라우저 개발자 도구 (F12) → Network 탭 → "Offline" 선택
3. 페이지 상태 확인

**기대 결과:**
- ✅ 상단에 "오프라인 상태입니다" 알림 표시
- ✅ 기존 데이터는 캐시에서 표시
- ✅ 새 제품 저장 시도 시 오류 메시지 표시

---

### 시나리오 10: PWA 설치 (모바일)

**목표:** PWA 설치 및 홈화면 추가 확인

**단계:**
1. 스마트폰 브라우저에서 배포 URL 접속
2. 주소창 우측 메뉴 → "홈 화면에 추가" 또는 "설치"
3. 앱 설치 완료 후 실행

**기대 결과:**
- ✅ 홈화면에 라이프컬처 아이콘 추가됨
- ✅ 앱으로 실행 시 풀스크린 모드
- ✅ 로그인 후 제품 정보 관리 가능

// F12 → Console 탭에서 다음 명령 실행

3. 상태 코드 200 확인

2. Local Storage → 현재 URL 선택
3. 저장된 데이터 확인
